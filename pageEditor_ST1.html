Home Page

Technical Description

The Home Page uses the shared HTML template with the standard <header>, <nav>, <main>, and <footer> structure. The main content area contains two primary sections: mission cards and an interactive content gallery.

HTML Structure: Three <div class="card"> elements display mission-related content, each containing an <img> with alt text, an <h3> heading, a descriptive <p>, and a <ul> with action highlights. The gallery section uses <a> elements styled as tiles that link to each student's content page.

CSS Techniques: Page-specific embedded CSS uses grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)) for responsive card layouts. Gallery tiles use position: relative with overlaid captions using position: absolute; bottom: 0 and a gradient background for text readability. Hover effects use transform: translateY(-6px) scale(1.02) with transitions for smooth animation.

Accessibility

All images include descriptive alt text. The navigation bar uses an aria-label="Main navigation" attribute. The active page is indicated visually via the .active CSS class. Semantic HTML elements (<header>, <nav>, <main>, <footer>, <section>) are used throughout for screen reader compatibility.

Gallery Page

Technical Description

The Gallery Page displays a grid of thumbnail images related to ocean conservation. The layout uses display: grid with auto-fit columns for responsiveness.

HTML Structure: Each gallery item is a <div class="gallery-item"> containing an <img>, a label, and data-* attributes storing the title, image source, and description for the modal view.

JavaScript Interactivity: Event listeners handle hover (mouseenter/mouseleave), click-to-expand (opening a modal), and close interactions (close button, click outside, Escape key). The modal is populated dynamically using getAttribute() and DOM manipulation. User customisation controls allow changing the colour scheme and font style of the extended view using select elements and JavaScript switch statements that modify style properties.

CSS Techniques: The modal uses position: fixed with flexbox centring. Gallery items have overflow: hidden with image transform: scale(1.08) on hover. An external CSS file (css/gallery.css) handles all page-specific styling.

Accessibility

All images have descriptive alt text. Gallery items include tabindex="0" for keyboard navigation. The modal has role="dialog" and aria-label. The close button includes aria-label="Close image view". Keyboard support allows Enter to open items and Escape to close the modal.

Content Page — Ocean Pollution

Technical Description

The content page covers ocean pollution with four structured sections using proper heading hierarchy (<h1> for page title, <h2> for sections). An internal navigation menu at the top provides anchor links (href="#section-id") to each section.

CSS Techniques: Embedded CSS styles the internal navigation with flexbox, content images with border-radius and box-shadow, and a fixed "Go to Top" button using position: fixed; bottom: 30px; right: 30px.

Accessibility

Four relevant images each have descriptive alt text. The internal navigation uses aria-label="Page sections". Heading hierarchy is logical and sequential. The "Go to Top" link includes an aria-label attribute.

Shared Template & Global CSS

Technical Description

The shared HTML template provides a consistent structure for all pages with a <header> containing the brand logo and navigation bar, a <main> content area, and a <footer> with links and student details. Comments within the HTML and CSS clearly indicate which sections must not be modified.

Global CSS: The external stylesheet (style.css) defines CSS custom properties (variables) for the colour palette, typography, spacing, and transitions. It includes shared styles for the header, navigation bar (with hover and active states), cards, buttons, forms, images, footer, and responsive breakpoints. Page-specific styles are embedded in each HTML file using <style> blocks. Google Fonts imports Open Sans for consistent typography.

The navigation bar uses flexbox with flex-wrap: wrap for responsiveness. The active page is indicated by the .nav-link.active class with a green background. All team members were instructed to only change the active class and their footer details.

Challenges and Lessons Learned

One significant challenge was ensuring the gallery modal worked correctly across different screen sizes. The initial implementation used fixed pixel values, which caused layout issues on mobile. Switching to max-width: 700px; width: 100% with viewport-relative padding resolved this.

Another challenge was coordinating the template with other team members. Clear comments in the HTML and CSS, along with a shared style guide document, helped everyone integrate their content without breaking the shared structure.

A key lesson learned was the importance of testing navigation links early. Several broken links were caught during group testing sessions, which reinforced the value of agreed file naming conventions.

Compliance with JANET Regulations

All pages comply with JANET regulations governing web page publication. Content is original and appropriate for an educational context. No offensive, discriminatory, or misleading material is included. All externally sourced images are royalty-free and properly referenced. The website does not collect or store personal data — the feedback form is client-side only and does not transmit information to any server.

The website follows responsible web publishing practices by using semantic HTML, providing alternative text for images, and ensuring content is accessible to users with disabilities.
