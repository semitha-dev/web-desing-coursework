Splash Screen

Technical Description

The Splash Screen serves as the first point of interaction. It does not use the shared template but matches the site's green colour palette and Open Sans typography.

HTML Structure: A full-viewport wrapper <div class="splash-wrapper"> contains a background image layer, a gradient overlay for readability, and a content area with the website title (<h1>), mission statement (<p class="mission">), team member list (<ul class="team-list">), loader animation, countdown, and skip button.

Auto-redirect: An HTML <meta http-equiv="refresh" content="4;url=home.html"> tag automatically redirects to the home page after 4 seconds, as specified in the coursework brief.

CSS Techniques: The background uses background: center/cover no-repeat for full-screen scaling. The overlay uses a linear-gradient with semi-transparent colours. The CSS loader animation uses @keyframes spin rotating a bordered circle 360 degrees. Flexbox centres all content vertically and horizontally. The skip button uses backdrop-filter: blur(4px) for a frosted glass effect.

JavaScript: A setInterval timer decrements a counter every second, updating the countdown text from 4 to 1. The skip button uses addEventListener('click') to clear the interval and immediately navigate via window.location.href.

Accessibility

The loader div includes aria-hidden="true" since it is decorative. Text contrast is ensured through the gradient overlay. The skip button is keyboard-focusable and clearly labelled.

SDG Action Impact Simulator

Technical Description

The AIS page uses the shared template and features 6 action cards that users can click to select sustainable actions related to SDG 14.

HTML Structure: Each card is a <div class="action-card"> with a data-points attribute. Cards contain an image, category label, title, description, and impact point display. A feedback summary area shows the running total.

JavaScript Interactivity: Click event listeners toggle a .selected class on cards. The updateScore() function iterates through all cards, sums points from selected ones, determines the impact level (Low: 0-5, Medium: 6-10, High: 11+), and updates the feedback area. Background images change dynamically via style.backgroundImage based on the current impact level. A reset button removes all .selected classes and re-runs the score calculation.

Embedded CSS: Over 6 CSS rules handle the card grid layout, selection state (border colour change and checkmark pseudo-element), card body styling, and feedback area presentation.

Accessibility

All card images have descriptive alt text. Selected cards have a visible green border and checkmark icon. The feedback area uses clear heading hierarchy. Impact level text is descriptive and not colour-dependent alone.

Content Page — Marine Protected Areas

Technical Description

This content page covers Marine Protected Areas with four sections, an internal navigation menu using anchor links, four images with alt text, and a fixed "Go to Top" button.

CSS Techniques: Embedded CSS provides internal navigation styling with flexbox and hover effects, image rounding with border-radius and shadows, and a fixed position "Go to Top" button.

Accessibility

Semantic headings follow <h1> to <h2> hierarchy. All images include descriptive alt text. The page navigation section uses aria-label.

Challenges and Lessons Learned

The countdown timer on the splash screen initially conflicted with the meta refresh — both would fire independently. I resolved this by keeping the meta refresh as the primary redirect mechanism (as required by the spec) and using JavaScript only for the visual countdown and skip functionality.

For the AIS, calculating impact scores required careful planning of the data model. Using data-* attributes on HTML elements kept the JavaScript clean and the HTML readable. I learned the importance of separating data from presentation logic.
