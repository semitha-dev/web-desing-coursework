Feedback & Programme Directory

Technical Description

The Feedback page combines three distinct sections: a scrollable programme directory, a ratings/reviews summary, and a JavaScript-enhanced feedback form.

HTML Structure: The scrollable section uses a wrapper with CSS max-height: 520px; overflow-y: auto containing 4 programme cards in a CSS Grid. Each card has an image, title, description, and call-to-action button. A styled dropdown allows sorting (non-functional as per spec). The reviews section displays an overall rating and 3 review cards with headings, descriptions, reviewer names, dates, and ratings.

Form Structure: Two <fieldset> groups with <legend> elements organise the form into "Personal Details" and "Feedback Details". Fields include text input, email, tel, textarea (with maxlength="500"), select dropdown, date, and datetime-local inputs as required by the specification.

JavaScript Interactivity: A live character counter updates on each input event on the message textarea. Form validation uses addEventListener('submit') with preventDefault(), checking each field and displaying field-specific error messages via classList.add('show'). Email validation uses a regex pattern. On successful validation, a confirmation message is shown and the form resets.

CSS: An external page-specific CSS file (css/feedback.css) styles the scrollable section, programme cards, review cards, form fields, error messages, and success confirmation. Custom scrollbar styling is applied for webkit browsers.

Accessibility

All form fields have associated <label> elements. Error messages appear near the relevant field. Images include alt text. Fieldset/legend grouping improves form comprehension for screen readers.

Team Page

Technical Description

The Team Page presents four team member blocks in a responsive grid layout using CSS Grid with auto-fit columns.

HTML Structure: Each member block is a <div class="team-member"> with tabindex="0" containing a thumbnail image, visible info (name and role), and a hidden details panel.

CSS Interactivity: The expandable details panel uses max-height: 0; overflow: hidden by default with a CSS transition. On :hover and :focus-within, the max-height changes to 250px, revealing the details with a smooth animation. Thumbnail images scale to 1.05 and reduce opacity on hover. The card itself translates upward and gains an enhanced shadow.

Accessibility

Each team member block includes tabindex="0" for keyboard access. The :focus-within selector ensures details expand on keyboard focus. All images have meaningful alt text. A visible focus state is present via the global CSS outline styles.

Content Page — The Overfishing Crisis

Technical Description

This content page explores overfishing through four structured sections with an internal navigation menu, four images, and a fixed "Go to Top" button. Embedded CSS handles page-specific styling including the content navigation, image presentation, and fixed-position button.

Accessibility

Logical heading hierarchy from <h1> to <h2>. All images include descriptive alt text. Internal navigation uses aria-label. Semantic elements used throughout.

Challenges and Lessons Learned

The live character counter was straightforward to implement but required careful handling of the maxlength attribute interaction. I learned that HTML's maxlength prevents additional input while JavaScript provides the visual feedback.

The team page hover expansion was initially jumpy due to fixed height values. Switching to max-height with transitions produced a much smoother result. This taught me the importance of choosing the right CSS property for animations.

Compliance with JANET Regulations

All content is original and educational. The feedback form does not transmit data to any server. No personal information is stored. All images are royalty-free and referenced. Content is appropriate and non-discriminatory.
