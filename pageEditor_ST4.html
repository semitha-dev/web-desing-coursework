User Profile Page

Technical Description

The User Profile page demonstrates JavaScript-driven interaction using prompt() to progressively collect user information and dynamically build a profile display.

HTML Structure: The page uses the shared template with a control area (Start, Reset, step buttons, revisit buttons), a progress indicator with a visual bar, and a profile output area with three hidden step sections revealed progressively.

JavaScript Implementation: A profileData object stores all user responses across three steps, each containing three prompts. Functions runStep1(), runStep2(), and runStep3() use prompt() to collect data step-by-step. Users can skip prompts by leaving them blank (null check and trim). The updateDisplay() function counts non-null values to calculate completion percentage, updates the progress bar width via style.width, shows/hides profile sections using classList.add('visible'), and displays a confirmation message at 100% completion.

User Control: Users can skip individual prompts or entire steps. Revisit buttons allow returning to any step to fill in previously skipped prompts. Skipped values display "(skipped)" in the output. The progress indicator adjusts dynamically — incomplete prompts reduce the completion percentage. A reset button clears all data and hides all output sections.

CSS: An external page-specific CSS file (css/profile.css) handles the progress bar gradient animation, step button states (completed/skipped), profile output sections with left border accents, and the confirmation section.

Accessibility

All visible content uses clear labels and descriptive text. Instructions explain the prompt-based workflow. The progress indicator provides both visual (bar) and text (percentage + count) feedback. Button labels are descriptive.

Sitemap Page (SVG)

Technical Description

The Sitemap page displays a graphical representation of the website hierarchy using inline SVG.

SVG Structure: The sitemap uses <svg viewBox="0 0 960 700"> for responsiveness. Nodes are represented as <rect> elements with rounded corners (rx="10") wrapped in <a> links to the corresponding pages. <line> elements connect parent and child nodes. <text> elements label each node.

Hierarchy: The sitemap shows: Splash Screen → Home Page → main pages (AIS, Gallery, Profile, Feedback, Team, Sitemap) → Content Pages (4) → Page Editors (4) → Validation Pages (4). Dashed lines distinguish content pages from main navigation pages.

Interactivity: Each node links to its corresponding page using SVG <a> elements. Hover effects change fill colour and stroke width via CSS. <title> elements inside each link provide accessible names for screen readers.

Responsiveness: The viewBox attribute combined with width: 100%; max-width: 1000px; height: auto in CSS ensures the SVG scales proportionally on all screen sizes.

Accessibility

The SVG has role="img" and aria-label. Each clickable node contains a <title> element. Links are keyboard-navigable via Tab. Focus states are styled with an outline. Text labels are visible for all nodes.

Challenges and Lessons Learned

The progressive profile builder required careful state management. Initially, I used separate variables for each field, but switching to a single profileData object made the code cleaner and the updateDisplay() function more maintainable.

The SVG sitemap required planning the coordinate system carefully within the viewBox. I initially used trial-and-error for positioning nodes but found that sketching the layout on paper first, then translating to coordinates, was far more efficient.

Handling the "skip" and "revisit" functionality was the most complex part. Ensuring that revisiting a step properly updated the profile without duplicating output required careful DOM manipulation and state tracking.

Compliance with JANET Regulations

All pages are original and educational. The profile page uses client-side prompt() only — no data is stored or transmitted to any server. All images are royalty-free and referenced. Content is appropriate and non-discriminatory. The SVG sitemap does not contain any external dependencies.
