Reflection on Validation

During validation, I initially encountered an error due to a duplicate id attribute on two navigation links. This was resolved by ensuring each element has a unique identifier. I also received a warning about the inline style attribute on one element, which I moved to the embedded CSS block for better separation of concerns.

The validation process reinforced the importance of checking HTML structure regularly during development rather than only at the end, as catching errors early is much simpler to fix.
